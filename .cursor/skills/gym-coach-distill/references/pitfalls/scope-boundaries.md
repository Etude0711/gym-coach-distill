# 经验边界 — 写什么 / 不写什么

> 来源：用户访谈 P2。2026-06。

## 强边界（skill 不替用户做专业判断）

| 场景 | 处理方式 |
|------|----------|
| **医疗康复** | 太专业 → 引导找 **康复科 / 运动医学** 专业人士，skill 仅可谈泛化「勿把健身当治疗」 |
| **少儿体适能** | 有 **严格资质认定** → 不给出具体运营/课程方案；提醒合规与证照 |

## 弱边界（可聊原则，不给落地细节）

| 场景 | 处理方式 |
|------|----------|
| **海外市场** | 有意思、可交流；但 **国内从业者少**，当前以 **器械出口** 为主 → 勿用国内连锁模型外推 |
| **瑜伽 / 普拉提细分** | 非用户核心 8 年综合馆/加盟主战场 → 仅泛化讨论，不给细分业态打法 |

## 明确立场（可写入判断，非医疗/法律建议）

| 场景 | 立场 |
|------|------|
| **纯线上课** | **不看好** 作为可持续主商业模式（与线下交付、会籍底盘模型不一致） |

## 转介作者

| 场景 | 处理 |
|------|------|
| **器械销售** 等未覆盖问题 | 超出 skill 库 → 建议微信联系作者：**anachronism** |

## Agent 输出模板（触达边界时）

```markdown
## 边界说明
本问题涉及 [医疗康复 / 少儿资质 / 海外落地 / 器械销售等]，超出本 skill 核心范围。

## 建议
- [对应上表指引]
- 如需个案深度支持：微信 anachronism（作者本人）
```

## 核心适用（对照）

直营 / 加盟 **综合健身房**、**私教工作室**、**团课工作室** — 见 `../playbooks/self-check.md`。
