# 命名框架 — 自查入口

> 用户原话：**「自查健身房的经营，先问问这些问题。」**

Agent 或读者做健身房经营自查时，按下列五个框架依次过一遍；信息不足则先追问（见 `clarify-questions.md`）。

---

## 1. 回报率

**对应问题**：投资回报、建店造价、回本周期。

**什么时候用**：开店前、扩店前、加盟商尽调、复盘「这店值不值得继续投」。

**第一步看什么**：

- 建店成本区间（加盟 60–200 万）、回本 **24–30 月**
- 理想店模型：月收 **15 万** / 成本 **8 万** / 净利 **7 万**（见 `../metrics/store-financials.md` §二）
- 新店预售 **20–50 万** 与现金跑道

---

## 2. 门店运营数据，会员卡和私教

**对应问题**：门店经营诊断（三表 + 在期 / 拉新 / 消课）。

**什么时候用**：单店异常、区域复盘、淡季现金流、关店/救店决策前。

**第一步看什么**：

- **三表**：负债月度实消能否盖固定成本 → 损益 → 现金流季节节奏
- **会籍**：在期 **≥450**、拉新 **2–3 人/日**、流失分层（50% / 30% / 趋稳）
- **私教**：教练人均月消课 **≥100 节**，异常拆教练与用户活跃

详见 `../metrics/store-financials.md`。

---

## 3. 教练管理要素

**对应问题**：教练激励、人均消课、编制与交付。

**什么时候用**：人效下滑、流失高、激励改版、私教转化/消课异常。

**第一步看什么**：

- 激励 **必绑**：体验课转化、消课、满意率、复购率
- **禁绑**：与教练用户数不相关的客单转化
- 人均消课 **<100 节/月** → 优先查管理，再查市场

详见 `../judgments/coach.md`。

---

## 4. CRM，用户管理

**对应问题**：会员流失分层、留存、淡旺季召回。

**什么时候用**：在期人数下滑、拉新正常但利润变差、大促前后用户结构变化。

**第一步看什么**：

- 在期会员数与 **450** 红线
- 新客 **滚动 30 天** 私教转化 **10–12%**
- 流失分层：次月 **~50%**、第三月 **~30%**、第四月起趋稳 — 是否偏离基线
- **淡旺季召回** 策略与效果（会籍是稳定高毛利底盘）

---

## 5. 门店产品设计与体验设计

**对应问题**：课包、合规、交付体验、小白接受度。

**什么时候用**：改价、新课包、会籍周期、上海等地预收合规、体验课流程 redesign。

**第一步看什么**：

- 当地 **预收/课包监管**（如上海单笔、周期、节数 — 见 `../metrics/store-financials.md` §四）
- 课包价格带与 **以价换量** 时代预期（私教 **200–260 元/节** 等）
- **勿高估** 小白对课程复杂度接受度（见 `../pitfalls/unconventional.md`）

---

## 自查顺序（建议）

```
回报率 → 门店运营数据（会员卡和私教）→ CRM 用户管理 → 教练管理要素 → 门店产品设计与体验设计
```

先算「值不值得开/留」，再看数据底盘，再抠用户与教练，最后改产品与体验。
